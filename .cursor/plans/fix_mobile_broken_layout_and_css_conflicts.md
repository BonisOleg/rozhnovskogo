# Fix mobile broken layout and CSS conflicts

**Проблема:** Мобільна версія має конфлікти в медіа-запитах між `layout.css`, `hero.css` та `mobile.css`. Деякі стилі дублюються, інші перекривають один одного некоректно (наприклад, позиціонування логотипу та бургер-меню).

**План дій:**

1. **Консолідація медіа-запитів:**
   - Перенести всі критичні мобільні фікси з `mobile.css` та `hero.css` у відповідні секції або уніфікувати їх.
   - Видалити застарілі згадки про `prozorro` та `land` у всіх CSS файлах.

2. **Оновлення [static/css/layout.css](static/css/layout.css):**
   - Виправити `header__inner` для мобілок: бургер зліва, лого по центру (абсолютно), дії справа.
   - Забезпечити, щоб `.parallax-section .container` на мобільних мав правильні відступи та не "вилітав".

3. **Оновлення [static/css/hero.css](static/css/hero.css):**
   - Виправити `.hero__inner` для мобілок: `padding-block: 2rem`, `max-width: 100%`.
   - Переконатися, що шрифт заголовка адаптується правильно.

4. **Оновлення [static/css/mobile.css](static/css/mobile.css):**
   - Видалити дублюючі стилі, які вже є в `layout.css`.
   - Залишити лише специфічні фікси для iOS Safari (safe-areas, 100svh).

5. **Видалення зайвого:**
   - `static/css/prozorro.css` — видалити файл, якщо він більше не потрібен.
